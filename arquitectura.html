1. Resumen Ejecutivo

AccesoEval es una herramienta web educativa que analiza pruebas escritas y sugiere adecuaciones de acceso para estudiantes con Necesidades Educativas Especiales (NEE), basándose en el Diseño Universal para el Aprendizaje (DUA).

El sistema detecta oportunidades de mejora en tres dimensiones:

● Comprensión Lectora ● Velocidad de Procesamiento ● Ansiedad Evaluativa

Principio fundamental: AccesoEval no modifica objetivos de aprendizaje, contenidos curriculares ni nivel cognitivo. Opera exclusivamente sobre accesibilidad, claridad lingüística, formato visual y estructura de instrucciones.

⚠️ Piloto MVP: La versión inicial trabaja exclusivamente con pruebas escritas (.docx). La arquitectura queda preparada para expandirse a otros tipos de instrumentos.
2. Recomendación: Word (.docx) vs Google Docs
✓ RECOMENDADO PARA MVP

📄 Opción B — Word (.docx)

✅ Formato universal entre docentes chilenos ✅ Sin dependencia de cuenta Google ✅ Funciona offline y en redes lentas ✅ Carga directa desde pendrive o escritorio ✅ Librería mammoth.js (extracción) o docx.js (regeneración) sin costos ✅ Mantiene formato original del docente ✅ Exporta documento adecuado con 1 clic ⚠️ Limitación: no edita en tiempo real sobre el archivo
FASE 2

🔗 Opción A — Google Docs

✅ Edición en tiempo real nativa ✅ Comentarios y control de cambios integrado ❌ Requiere cuenta Google institucional de cada docente ❌ Necesita configuración OAuth 2.0 (semanas de dev) ❌ Dependencia de conectividad estable ❌ No todos los colegios tienen G Suite ❌ Costo: Google Workspace Education Plus (~$3 USD/usuario/mes) ❌ Datos en servidores de Google (privacidad)

Veredicto técnico: Para el MVP, Word (.docx) es más eficiente, accesible y de menor riesgo técnico. La plataforma exporta el documento adecuado como nuevo .docx, preservando la compatibilidad total. La integración Google Docs puede agregarse en Fase 2 como opción adicional.

3.2 Componentes clave

📥 Ingesta de documento

mammoth.js extrae texto estructurado del .docx manteniendo párrafos, encabezados y negrita. El texto se envía al motor de análisis.

🔍 Motor de análisis

Compara el texto contra matrices de criterios JSON (cargables). Produce un listado de segmentos problemáticos con posición, categoría y sugerencia.

✏️ Editor interactivo

Renderiza el texto con highlights de colores. Cada highlight enlaza con una tarjeta de sugerencia. Aceptar/rechazar actualiza el estado en tiempo real.

📤 Exportador

docx.js regenera el archivo .docx aplicando los cambios aceptados. El docente descarga la versión adecuada lista para imprimir o compartir.

🤖 IA (Fase 2)

Claude API (Anthropic) analiza matices lingüísticos que las matrices no detectan. Genera sugerencias contextualizadas y justificaciones enriquecidas.

🔐 Autenticación

Login por correo institucional + JWT. La BD institucional carga automáticamente docente, asignatura, cursos y nivel al autenticarse.

6. Modelo de Análisis Automático

El motor de análisis funciona en dos modos complementarios, diseñados para escalar desde el MVP hasta producción completa:

Modo 1 — Basado en matrices (MVP)

Compara el texto contra criterios predefinidos en archivos JSON. Sin costo de IA. Funciona offline.

// Estructura de matriz de criterios (criteria.json) { "version": "1.0", "idioma": "es", "criterios": [ { "id": "CL-001", "categoria": "comprension_lectora", "subcategoria": "Vocabulario técnico", "patron": "orden jerárquico|precedencia|operatoria", "tipo_patron": "regex", "sensibilidad": "intermedio", "nivel_educativo": ["basico"], "sugerencia": "Simplificar vocabulario o definir entre paréntesis", "justificacion": "DUA Pauta 2.1 — CAST 2018", "evidencia": "Muy alta" } ] }
Modo 2 — IA aumentada (Fase 2)

Claude API procesa el texto completo en contexto, detecta problemas que las regex no capturan (ironía, ambigüedad implícita, tono) y genera sugerencias personalizadas según asignatura y nivel.

Criterios predefinidos (muestra)
CL-001 · Vocabulario técnico acumulado

Patrón: 3+ términos técnicos consecutivos sin definición. Acción: señalar cadena, sugerir glosario en pie de ítem.

CL-002 · Instrucciones con subordinadas múltiples

Patrón: oraciones con más de 2 cláusulas subordinadas. Acción: dividir en instrucciones cortas numeradas.

VP-001 · Instrucciones en bloque sin estructura visual

Patrón: párrafo con más de 3 instrucciones sin viñetas. Acción: convertir a lista numerada.

VP-002 · Tiempo sin distribución por sección

Patrón: mención de tiempo total sin desglose. Acción: agregar tabla de tiempo sugerido por ítem.

AE-001 · Lenguaje prohibitivo

Patrón: "no se aceptará", "será sancionado", "queda prohibido". Acción: reformular en positivo.

AE-002 · Consecuencias negativas explícitas

Patrón: "de lo contrario", "si no", "será inválido". Acción: reformular orientado a puntaje parcial.

🔑 Dato clave para el cliente: Todo el sistema está diseñado para que las matrices de criterios sean el corazón del análisis. Cuando entregues tu documento de criterios (categorías, subcategorías, patrones, sugerencias, justificaciones y niveles de evidencia), se carga como JSON y el motor lo usa inmediatamente, sin modificar el código.
